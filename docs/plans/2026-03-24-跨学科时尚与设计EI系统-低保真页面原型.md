# 跨学科时尚与设计 EI 系统：低保真页面原型

## 1. 说明

这份文档不是视觉稿，而是**低保真页面原型说明**。

你可以把它理解成：

- 页面应该放哪些区域
- 用户先看到什么，后操作什么
- 哪些按钮最重要
- 哪些信息必须一直可见

第一版先聚焦 5 个最核心页面：

1. 项目首页
2. 新建论文项目
3. 研究画像页
4. 论文提纲页
5. 章节写作页

## 2. 全局布局规范

建议整个系统统一采用三栏思路，但不是每页都必须三栏。

### 全局布局

- 左侧：主导航
- 中间：主工作区
- 右侧：辅助信息区或操作提示区

### 左侧主导航建议

- 项目首页
- 当前项目
- 会议与模板
- 研究画像
- 论文提纲
- 章节写作
- 参考文献
- 图文材料
- 检查与导出
- 版本记录

### 顶部全局栏建议

- 产品名称
- 当前项目名称
- 自动保存状态
- 全局搜索
- 用户入口

### 全局交互原则

- 任何时刻都让用户知道“我现在在哪一步”
- 任何重要生成动作都要显示状态
- 任何高风险内容都要能一眼看到提醒

## 3. 页面一：项目首页

### 页面目标

让用户快速进入项目，而不是一上来就被复杂表单吓住。

### 低保真结构

```text
+----------------------------------------------------------------------------------+
| Logo / 产品名                     搜索项目                           用户头像     |
+----------------------------------------------------------------------------------+
| 左侧导航         | 欢迎区：继续你的论文工作                                         |
| - 项目首页       | “从主题到投稿草稿，按步骤完成”                                   |
| - 新建项目       | [ 新建论文项目 ] [ 查看示例项目 ]                                |
|                  +----------------------------------------------------------------+
|                  | 最近项目                                                        |
|                  | +------------------+ +------------------+ +------------------+ |
|                  | | 项目卡片         | | 项目卡片         | | 项目卡片         | |
|                  | | 题目             | | 题目             | | 题目             | |
|                  | | 当前阶段         | | 当前阶段         | | 当前阶段         | |
|                  | | 目标会议         | | 目标会议         | | 目标会议         | |
|                  | | [继续]           | | [继续]           | | [继续]           | |
|                  | +------------------+ +------------------+ +------------------+ |
|                  +----------------------------------------------------------------+
|                  | 最近活动 / 系统提示                                             |
|                  | - 哪些项目缺会议模板                                            |
|                  | - 哪些项目还没完成提纲                                          |
+----------------------------------------------------------------------------------+
```

### 页面重点

- 把“继续上次工作”放到最醒目的位置
- 把“新建项目”作为主要入口
- 每个项目卡片只展示最关键的 3 个信息：题目、阶段、会议

### 页面状态

- 空状态：展示引导文案和示例项目
- 正常状态：展示项目卡片
- 风险状态：项目卡片显示“缺模板”“缺提纲”“待检查”

## 4. 页面二：新建论文项目

### 页面目标

把用户模糊的想法转成结构化项目。

### 推荐做成分步表单

不要让用户一次填完整页，建议拆成 4 步。

1. 研究主题
2. 会议与方向
3. 材料与方法
4. 确认创建

### 低保真结构

```text
+----------------------------------------------------------------------------------+
| 返回项目首页                     新建论文项目                         步骤 1/4      |
+----------------------------------------------------------------------------------+
| 左侧步骤条       | 主内容区                                                     |
| 1 研究主题       | 标题：先告诉我你想研究什么                                    |
| 2 会议与方向     |                                                              |
| 3 材料与方法     | 论文主题 [_______________________________________________]     |
| 4 确认创建       | 关键词   [________________] [________________] [___________]   |
|                  | 研究对象 [_______________________________________________]     |
|                  |                                                              |
|                  | 已有想法说明                                                 |
|                  | [                                                          ] |
|                  | [                                                          ] |
|                  |                                                              |
|                  | [ 系统帮我判断题目类型 ]                                      |
|                  |                                                [下一步]       |
+----------------------------------------------------------------------------------+
```

### 第二步建议字段

- 目标会议
- 模板类型
- 学科主类
- 学科副类
- 输出语言

### 第三步建议字段

- 是否已有参考文献
- 是否已有图片/草图/作品
- 是否已有问卷/访谈/数据
- 预期方法
- 上传附件

### 页面设计重点

- 每一步只做一类事情
- 每一步都告诉用户“为什么要填这个”
- 提供“我不确定，系统帮我判断”

## 5. 页面三：研究画像页

### 页面目标

在正式写作前，先把“研究类型”和“推荐写法”讲清楚。

### 低保真结构

```text
+----------------------------------------------------------------------------------+
| 当前项目：xxx                           研究画像                                  |
+----------------------------------------------------------------------------------+
| 左侧导航         | 主工作区                                  | 右侧提示区          |
|                  | +----------------------------------------+ | 高风险提醒         |
|                  | | 研究类型判断                           | | - 缺用户样本      |
|                  | | 设计实践 + 用户研究                    | | - 缺正式文献      |
|                  | | 置信度：高                             | | - 方法描述过弱    |
|                  | +----------------------------------------+ +--------------------+
|                  | +----------------------------------------+ | 推荐补充材料       |
|                  | | 推荐研究方法组合                       | | - 访谈提纲         |
|                  | | 1. 设计实践                            | | - 用户测试记录     |
|                  | | 2. 用户测试                            | | - 作品图           |
|                  | | 3. 结果分析                            | +--------------------+
|                  | +----------------------------------------+
|                  | +----------------------------------------+
|                  | | 推荐论文结构                           |
|                  | | 问题定义 -> 设计目标 -> 设计方案       |
|                  | | -> 原型展示 -> 用户测试 -> 讨论        |
|                  | +----------------------------------------+
|                  |                      [接受推荐方案] [重新判断] |
+----------------------------------------------------------------------------------+
```

### 页面重点

- 不只是告诉用户“你是什么类型”
- 还要告诉用户“为什么这样判断”
- 更重要的是告诉用户“接下来应该怎么写”

### 页面核心操作

- 接受当前方案
- 手动切换研究类型
- 补充缺失材料
- 进入提纲生成

## 6. 页面四：论文提纲页

### 页面目标

先把全篇骨架定下来，避免后面反复推翻。

### 低保真结构

```text
+-------------------------------------------------------------------------------------------+
| 当前项目：xxx                                      论文提纲                                |
+-------------------------------------------------------------------------------------------+
| 左侧章节树                | 中间提纲编辑区                           | 右侧建议区          |
| - 标题                    | 题目候选                                 | 结构建议            |
| - 摘要                    | 1. xxxxxxxxxxxxxxxxx                    | - 当前结构适合      |
| - 引言                    | 2. xxxxxxxxxxxxxxxxx                    |   混合方法论文      |
| - 相关研究                | [换一组题目] [采用该题目]               |                     |
| - 方法                    +-----------------------------------------+ 风险提醒            |
| - 结果                    | 摘要方向                                 | - 结果部分偏弱      |
| - 讨论                    | [摘要思路说明...]                        | - 缺案例来源        |
| - 结论                    +-----------------------------------------+                     |
|                           | 章节结构                                 | 写作提醒            |
|                           | 1 引言                                   | - 方法章要写样本    |
|                           | 2 理论与相关研究                         | - 讨论章别写成结论  |
|                           | 3 设计方案与实现                         |                     |
|                           | 4 用户测试与分析                         |                     |
|                           | 5 讨论                                   |                     |
|                           | 6 结论                                   |                     |
|                           | [新增章节] [调整顺序] [锁定提纲]         |                     |
+-------------------------------------------------------------------------------------------+
```

### 页面重点

- 左边是结构树，给方向感
- 中间是可编辑内容
- 右边持续给建议，不要把建议藏起来

### 页面关键操作

- 生成题目候选
- 生成摘要方向
- 编辑章节名称
- 增删章节
- 为每章写“章节目标”
- 锁定提纲版本

## 7. 页面五：章节写作页

### 页面目标

这是全系统最重要的页面，必须围绕“逐章生成、逐段修改、随时看上下文”来设计。

### 低保真结构

```text
+------------------------------------------------------------------------------------------------+
| 当前项目：xxx / 当前章节：方法与设计实现                                  自动保存中...        |
+------------------------------------------------------------------------------------------------+
| 左侧章节目录            | 中间正文编辑区                                  | 右侧辅助面板       |
| - 标题                  | 章节标题：方法与设计实现                         | 章节目标            |
| - 摘要                  | [ 生成本章草稿 ] [ 只重写这一段 ] [ 扩写 ]       | - 说明设计目标      |
| - 引言                  | [ 压缩 ] [ 更学术 ] [ 插入引用 ] [ 插入图注 ]    | - 交代实施步骤      |
| - 理论                  +------------------------------------------------+ - 写清评价方式      |
| - 方法                  | 本章正文编辑区                                   +--------------------+
| - 结果                  | [                                                ]| 已绑定引用          |
| - 讨论                  | [                                                ]| [1] xxx             |
| - 结论                  | [                                                ]| [2] xxx             |
|                         | [                                                ]+--------------------+
|                         | [                                                ]| 已绑定图文材料      |
|                         +------------------------------------------------+ | - 原型图 3张       |
|                         | 段落评论 / 风险标记                              | - 用户测试图表      |
|                         | - 第 2 段：缺引用                                +--------------------+
|                         | - 第 4 段：需人工确认实验结果                    | 风险提醒            |
|                         +------------------------------------------------+ | - 结论支撑不足      |
|                         | [保存本章] [标记已确认] [进入下一章]             | - 样本量未说明      |
+------------------------------------------------------------------------------------------------+
```

### 页面重点

- 中间区域必须足够大，因为这是主战场
- 右侧面板不能只放装饰信息，必须放真的有用的东西
- 所有 AI 动作尽量做成“局部操作”，别让用户每次整章重来

### 页面核心能力

- 按章节生成
- 按段落改写
- 引用绑定
- 图文材料绑定
- 风险标记
- 章节完成状态切换

## 8. 页面六：检查与导出页

### 页面目标

让用户在导出前知道“哪里还不行”。

### 低保真结构

```text
+----------------------------------------------------------------------------------+
| 检查与导出                                                                      |
+----------------------------------------------------------------------------------+
| 总体状态：建议修改                                                               |
|                                                                                  |
| +----------------------+ +----------------------+ +----------------------------+ |
| | 结构检查             | | 引用检查             | | 格式检查                   | |
| | 1 个必须修改         | | 2 个建议修改         | | 已通过                     | |
| +----------------------+ +----------------------+ +----------------------------+ |
|                                                                                  |
| 问题列表                                                                           |
| - 方法章缺少样本来源说明                                                          |
| - 第 3 章第 2 段缺参考文献                                                        |
| - 结论与摘要贡献点不完全一致                                                      |
|                                                                                  |
| 导出区                                                                             |
| [导出 LaTeX] [导出 DOCX] [导出 PDF 预览]                                          |
+----------------------------------------------------------------------------------+
```

### 页面重点

- 先展示问题，再展示导出按钮
- 问题要分“必须修改”和“建议修改”
- 导出动作要简单直接

## 9. 关键组件清单

为了后面更容易开发，第一版建议优先沉淀下面这些通用组件：

1. 项目卡片
2. 步骤条
3. 状态标签
4. 风险提示卡
5. 章节树
6. 右侧辅助面板
7. 文献列表
8. 图文材料卡片
9. 检查结果列表
10. 导出按钮组

## 10. 关键状态说明

### 10.1 生成中

- 按钮变为禁用
- 显示“正在生成提纲/章节，请稍等”
- 保留上一次内容，不清空编辑区

### 10.2 内容为空

- 显示引导动作
- 明确告诉用户先点哪个按钮

### 10.3 检查失败

- 显示失败原因
- 告诉用户应该去哪个页面修

### 10.4 已完成

- 用明确标签显示
- 提供“继续下一步”按钮

## 11. 第一版最值得先画出来的 3 张图

如果你现在就要去画线框图，优先画这 3 张：

1. 新建论文项目页
2. 论文提纲页
3. 章节写作页

原因：

- 这 3 页决定了产品核心体验
- 也是后续开发最重要的 3 个页面

## 12. 下一步建议

如果继续推进，最顺的顺序是：

1. 先把这 5 个页面画成正式线框图
2. 再补参考文献页和图文材料页
3. 然后定义组件库和前端路由
4. 最后进入开发

一句话总结：

**第一版页面原型应围绕“项目推进感、结构清晰、局部可改、风险可见”这 4 个原则来设计。**
